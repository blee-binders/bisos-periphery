*** File location convention
- Every panel lives at =<node>/_nodeBase_/fullUsagePanel-en.org=.
- The git working tree (=/bisos/git/auth/bxRepos/blee-binders/bisos-periphery/...=) is
  surfaced under =/bisos/panels/bisos-periphery/...= (a symlink/overlay). Panels
  reference BOTH paths: =This File::= uses the =/bisos/panels/= path, and a
  =/file-truename:/= line gives the real git-tree path.
- Directory nesting = the panel hierarchy (parent / descendents / siblings / ancestors).

*** Anatomy --- everything is a Dynamic Block (=####+BEGIN: <type> ... ####+END=)
In order, each panel contains:
 1. =bxPanel:common/title-plus= --- =#+title:= (node path), =#+roam_tags: branch=,
    =#+roam_key:=, and a =[[roam:...]]= link to the parent.
 2. =blee:bxPanel:topPanelControls= --- standard control/RipGrep/Maintenance bar;
    includes =This File ::= (panels path) and a =/file-truename:/= (git-tree path).
 3. =blee:bxPanel:title= (=:title "auto"=) --- the FPF/ByStar banner with =~Node:: <parent> -- <leaf>~=.
 4. =blee:bxPanel:terseTreeNavigator= --- auto-generated Decedents / Siblings / Ancestors nav links.
 5. =blee:bxPanel:foldingSection :outLevel 1 :title "Overview" :anchor "Panel"= ---
    the first content section; placeholder body is "Overview Comes Here."
 6. (content) additional =foldingSection= blocks for the actual material, optionally
    with =:extraInfo= (a short right-margin tag like "Howto"). Runnable steps use
    =blee:bxPanel:runResult :command "..."= blocks (see outlook for working examples).
 7. =blee:bxPanel:separator=
 8. =blee:bxPanel:evolution= + =foldingSection "Notes, Ideas, Tasks, Agenda"= (holds
    =*** TODO= items) + =blee:bxPanel:evolutionMaintainers= (Problem Report / Maintainers).
 9. =blee:bxPanel:footerPanelControls=, =footerOrgParams= (STARTUP/SEQ_TODO/TAGS/=#+CATEGORY: N:<leaf>=),
    and =footerEmacsParams= (the Local Variables block).
